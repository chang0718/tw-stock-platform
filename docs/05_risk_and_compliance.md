# 05 — 風險與合規

## 平台合規邊界

### 允許
- 提供量化評分作為篩選工具
- 顯示歷史行情與基本面數據（標示來源）
- 說明因子分析邏輯
- 協助建立風險評估流程
- 提醒估值過高、波動風險

### 不允許
- 保證任何特定報酬
- 提供個人化買賣建議（未持牌）
- 使用或傳播內線消息
- 鼓勵槓桿追漲、頻繁當沖
- 捏造財報數字或分析師目標價

## 資料風險

| 風險 | 說明 | 緩解方式 |
|------|------|--------|
| API 回傳空資料 | 非交易日 / 盤中 TWSE 回傳 [] | 顯示 "--"，不顯示假值 |
| FinMind 超流量 | 免費版有每日請求上限 | 自動切換 yfinance 備援 |
| SSL 憑證問題 | 台灣政府 API 憑證缺少 SKI | verify=False + 抑制警告 |
| Streamlit Cloud 重啟 | 所有 JSON 資料消失 | 本機執行不受影響；雲端需外部 DB |
| yfinance 延遲 | 資料延遲 15 分鐘 | 標示資料來源，不用於即時交易 |

## 免責聲明模板

平台所有分析頁面底部應顯示：

> ⚠️ 本平台資料僅供研究參考，不構成投資建議。
> 投資有風險，決策前請自行評估風險承受度，必要時諮詢合格專業人士。
> 歷史績效不保證未來報酬。

## SSL 處理說明

台灣政府 API（TWSE/TPEx）使用的 SSL 憑證缺少「Subject Key Identifier」擴充，
導致 Python 3.12+ 的 requests 拒絕連線。

解決方案：在 `utils.get_retry_session()` 設定 `session.verify = False`，
並使用 `urllib3.disable_warnings()` 抑制 InsecureRequestWarning。

此為台灣政府開放資料 API 的已知問題，業界通用解法。
連線仍使用 HTTPS，只是略過憑證格式驗證。
